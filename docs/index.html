Provalo subito

SOLEM è in fase di sviluppo attivo. Non c'è ancora ISO ufficiale hostata. Per provare:

Opzione A — Build VM in WSL/Linux (richiede Nix)

# Su Linux/WSL con Nix installato:
curl --proto '=https' --tlsv1.2 -sSf https://nixos.org/nix/install | sh

# Clone repo
git clone https://github.com/rguidotti-design/solem
cd solem

# Build + run VM
nix build .#vm
./result/bin/run-solem-vm-vm

# Si apre QEMU. Login: user=gavio pass=gavio
# Dentro la VM:
solem-welcome              # wizard primo setup
solem-localhost            # dashboard endpoint
solem-demo                 # walkthrough 10 capability

Opzione B — Build ISO + boot in VirtualBox

# Su Linux/WSL:
git clone https://github.com/rguidotti-design/solem
cd solem
nix build .#iso

# Trova la ISO:
ls result/iso/*.iso

# Crea VirtualBox VM:
# - Type: Linux
# - Version: Other Linux 64-bit
# - RAM: 4GB+
# - Storage: 20GB+
# - Mount ISO come live CD
# - Boot → installer Calamares

Opzione C — Solo leggi il codice

git clone https://github.com/rguidotti-design/solem
cd solem
ls nixos/modules/    # 100+ moduli SOLEM
cat docs/GAPS-VERO-OS.md   # analisi onesta cosa manca

Download ISO

⚠️ ISO non ancora hostata su CDN ufficiale. Per ora la build è locale via comandi sopra. Hosting ISO + GPG signing in roadmap (Step 50 scaffolding pronto).

Quando disponibile, install Calamares: boot ISO → doppio-click "Installa" → 5-15 min → reboot.

Layer di sicurezza (selezione)

1-3. Isolamento AI

UID 970 dedicato, nftables egress UID-based, systemd hardening override (NoNewPrivileges, caps drop, syscall filter).

4. Canary Honey Tokens

File esca in /etc/solem/canary/. Qualsiasi lettura → kill GAVIO immediato + notify desktop critical.

6, 10. Kernel hardening

Sysctl strict (BPF, ptrace, lockdown), linuxPackages_hardened (KSPP), module blacklist (cramfs, dccp, firewire, ...).

7. DNS allowlist

Unbound locale con local-zone "." refuse. Solo domini whitelist risolvono. Anti DNS tunneling.

8. AppArmor MAC

Profilo kernel-enforced per /var/lib/gavio-ai/venv/bin/python3. Deny esplicito /etc/shadow, /home/gavio, /dev/mem.

22-23. Self red-team + heal

Daemon notturno attacca il proprio sistema, identifica buchi, applica fix safe automatici. Auto-improving.
